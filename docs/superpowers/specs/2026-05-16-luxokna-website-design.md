# LuxOkna — дизайн сайта

**Дата:** 2026-05-16
**Заказчик:** LuxOkna, Грозный (Чечня) — премиум-окна Schüco / Rehau / Alutech
**Цель:** многостраничный сайт-визитка с калькулятором-лидогеном, отдающим заявки в WhatsApp

---

## 1. Цели и не-цели

**Цели:**
- Премиум-восприятие, соответствующее уровню брендов Schüco / Rehau / Alutech.
- Лидоген через калькулятор → deeplink в WhatsApp `wa.me/79288983897` с заполненным сообщением.
- 8 страниц с SEO-оптимизацией под локальные низкочастотные запросы (например, «алюминиевые витражи Грозный», «Schüco в Чечне»).
- Дизайн, который сознательно избегает визуальных штампов AI-сгенерированных сайтов — выглядит как работа студии.

**Не-цели (YAGNI):**
- Админ-панель / CMS.
- E-commerce (корзина, оплата).
- Аутентификация пользователей.
- Многоязычность (i18n).
- Блог.
- Dark mode (ломает тёплую bone-палитру).
- Серверная логика лидов (CRM-интеграции, email-уведомления).

## 2. Стек

| Слой | Решение | Обоснование |
|---|---|---|
| Фреймворк | Next.js 15 (App Router) + TypeScript strict | SSG + SEO + хороший DX |
| Стилизация | **Vanilla Extract** (zero-runtime CSS-in-TS) | Уход от Tailwind-стереотипа, типобезопасные токены |
| Анимация | **GSAP + ScrollTrigger**, **Lenis** (smooth scroll), **View Transitions API** | Не Framer Motion (overused в AI-выводе) |
| Типографика | Fraunces Variable (display serif), Switzer Variable (sans), JetBrains Mono Variable | Не Inter |
| Иконки | Кастомные SVG в `components/icons/` | Не Lucide |
| Изображения | `next/image` + AVIF/WebP + `plaiceholder` для LQIP | Лучшая производительность и плавная загрузка |
| Контент | TS-файлы в `content/` | Без CMS, заказчик пришлёт материалы позже, мы подменяем эти файлы |
| Тесты | Vitest + Testing Library, 1 Playwright smoke | Покрываем калькулятор |
| Деплой | Vercel | Бесплатный hobby-план, нативный для Next.js |
| Аналитика | Yandex.Metrika + GA4 | Россия + резерв |

## 5. Дизайн-система

### Цвета
- `bone` `#F2ECDF` — основной фон
- `boneDeep` `#E8DFCB` — карточки/разделители
- `char` `#1A1714` — основной текст
- `charSoft` `#3A332C` — вторичный текст
- `copper` `#A85031` — главный акцент (CTA, ошибки)
- `copperDeep` `#7D3A22` — hover акцента
- `sand` `#7A6A52` — подписи, мета
- `rule` `rgba(26, 23, 20, 0.12)` — hairline-линии
- `paper` `#FAF6EC` — карточки в карточках

Контраст `char` на `bone` = 14.2:1 (AA+).

### Типографика
- Display: **Fraunces Variable** (serif, выразительный), `font-variation-settings: "SOFT" 50, "WONK" 1`, дроп-капы через `::first-letter`.
- Body: **Switzer Variable** (grotesk), weight 400, letter-spacing `-0.005em`.
- Mono: **JetBrains Mono Variable**, uppercase, weight 500, tracking `+0.05em` — для меты, дат, размеров.
- Большие числа в калькуляторе: Fraunces с `"opsz" 144`.

Размеры (editorial-шкала, не tailwind): `micro 11 · meta 13 · body 16 · lead 19 · h4 24 · h3 32 · h2 48 · h1 72 · display 120`.

### Пространство
Шкала: `4 · 8 · 12 · 16 · 24 · 32 · 48 · 64 · 96 · 128 · 192`.
Радиус: только `0 / 2px / 6px` — никаких `2xl`-овалов.
Линии: `hair 1px / thin 1.5px / bold 3px`.

### Сетка
12 колонок, gutter 24px, max-width 1440px, боковая полоса 64px.
Используется **асимметрично**: текст в 5 колонках, фото на 8 колонок выезжает за правый край.

### Компоненты-примитивы
- `<Hairline />` — 1px-линия `rule`-цвета. Вместо `border-radius`-карточек.
- `<SectionNumber n="01" title="Окна" />` — большая моно-цифра + название.
- `<Marquee items={[]} />` — бесконечная маркиза (GSAP).
- `<MagneticButton />` — magnetic-эффект на курсор.
- `<RevealImage />` — clip-path inset reveal в viewport.
- `<Cursor />` — кастомный курсор-«окошко» (24×24, hairline border), magnet к `[data-cursor]`.

### Иконки
Все нарисованы под одну линейную систему: viewBox 24×24, stroke 1.25px, скруглённые концы. Минимальный набор: WhatsApp, Instagram, стрелки (←↑→↓), плюс/минус, чек, крест, фильтр, телефон, локация.

### Брейкпоинты
`mobile <640 · tablet 640–960 · desktop 960–1280 · wide ≥1280`.
На мобиле: editorial-сетка → один столбец, боковая полоса скрыта, шторка-калькулятор → fullscreen.

## 6. Калькулятор-лидоген

**Форма:** боковая шторка 480px справа (на мобиле — fullscreen). Открывается по кнопке «Рассчитать проём» с любой страницы.

**Точки входа:** плавающая магнитная кнопка снизу справа (везде), CTA в hero главной, отдельная **секция-презентация на главной** (`/`) и на `/kontakty` — крупный блок с превью полей и одной кнопкой, которая открывает ту же шторку. Шторка-калькулятор реализована один раз как компонент `<Calculator />`, секции — её триггеры.

**View Transitions:** применяем для **service-card → service-page** (общая hero-фотография), для **object-thumb → object-lightbox** в галерее `/obyekty`, и для общего `cross-fade` между страницами по умолчанию.

**Шаги (4):**

1. **Проём** — ширина (мм), высота (мм), числовые инпуты со степпером ±50 мм. Под полями live-расчёт площади (м²). Валидация: диапазон 300–6000 мм.
2. **Материал** — две карточки: «Алюминий» / «ПВХ» с подписью (холодный/тёплый) и SVG-сечением профиля.
3. **Сегмент** — три карточки:
   - `Alutech` — премиум
   - `Schüco / Rehau` — премиум/средний
   - Эконом
   Под каждой — одна строка преимуществ.
4. **Контакт** — имя (опционально), телефон (обязательно, маска `+7 (___) ___-__-__`, валидация через `libphonenumber-js`). Под полями — **редактируемое превью** сообщения для WhatsApp.

**Финал:** кнопка `Открыть в WhatsApp →`, ведёт на `https://wa.me/79288983897?text=<urlencoded message>`. Шаблон сообщения:

```
Здравствуйте! Хочу рассчитать остекление:

• Проём: 1 200 × 1 400 мм (1.68 м²)
• Материал: ПВХ
• Сегмент: Schüco / Rehau (премиум/средний)
• Имя: Саид
• Телефон: +7 (988) 123-45-67

Отправлено через luxokna.ru
```

**Доступность:**
- Focus trap внутри шторки.
- Escape закрывает.
- ARIA-роль `dialog`, `aria-labelledby`, `aria-modal="true"`.
- Tab-навигация между шагами.
- `inputmode="numeric"` на числовых полях.

**Аналитика:**
- `calc_open` — открытие шторки.
- `calc_step_2/3/4` — переход на каждый шаг.
- `wa_open` — клик финальной кнопки.
- `wa_floating_click` — клик плавающей кнопки.

**Состояние:** хранится в `useReducer` внутри компонента, без глобального стора. Восстанавливается из `sessionStorage` при случайном закрытии шторки.

**Что НЕ показываем в калькуляторе:** цены — ни вилки, ни «от X ₽». Только сбор лида.

## 7. Контент-модель

Все данные в `content/*.ts` как типизированные структуры:

- `company.ts` — название, город, годы на рынке, WA, Instagram, бренды.
- `services.ts` — массив из 4 услуг: slug, номер, title, tagline, материалы, бренды, секции, hero-изображение.
- `objects.ts` — портфолио: `{ slug, title, city, year, service, area (м²), brand, segment, images, description? }`.
- `reviews.ts` — отзывы: автор, город, текст, рейтинг, источник.
- `process.ts` — 5 шагов от заявки до монтажа.
- `brands.ts` — Schüco / Rehau / Alutech: лого, цвет акцента, преимущества, ссылка.
- `faq.ts` — общий FAQ + per-service.
- `seo.ts` — title/description/og-image по маршрутам.

На старте — плейсхолдеры с разумными значениями + реальными фактами (20 лет, бренды, WA). При получении материалов от заказчика подменяем только эти файлы.

## 8. SEO и производительность

**SEO:**
- `generateMetadata()` per page из `content/seo.ts`.
- Шаблон title: `{page} — LuxOkna · Schüco / Rehau / Alutech · Грозный`.
- OG-image 1200×630 — через Next.js OG-image route, из hero + текста.
- `sitemap.ts` и `robots.ts` в `app/`.
- JSON-LD `LocalBusiness` в `layout.tsx` (адрес, телефон, координаты 43.3168°N 45.6981°E, `priceRange: "₽₽₽"`).
- JSON-LD `Service` на страницах услуг.
- Canonical URLs.
- Один H1 на страницу, иерархия H2/H3, `lang="ru"`.

**Производительность (цель Lighthouse ≥ 95):**
- Vanilla Extract — zero-runtime, CSS извлекается в `.css` при сборке.
- AVIF + WebP fallback, `priority` только на hero, остальные lazy, `placeholder="blur"`.
- Шрифты через `next/font/local`, `display: swap`, variable .woff2 (один файл на семейство), subset RU+latin.
- GSAP/Lenis — `next/dynamic` с `ssr: false`.
- Preload только hero-картинки главной.
